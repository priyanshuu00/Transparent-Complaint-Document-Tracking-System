## Round 2 Updates

### 1. System Improvements

#### 1.1 User Roles and Access
- Add clear role-based access so citizens, officials, and administrators can only perform actions allowed for their role.

#### 1.2 Time-Bound Workflow
- Define fixed time limits for each workflow stage.
- Automatically mark requests as overdue when time limits are crossed.

#### 1.3 Escalation Mechanism
- Escalate requests that face repeated or long delays to higher authorities for action.

#### 1.4 QR-Based Verification
- Improve QR verification by including issuing authority details and completion date and time.

#### 1.5 Audit Logs
- Maintain complete audit logs for all actions, status updates, and responsibility changes.

---

### 2. Documentation Improvements

#### 2.1 Clear Problem Statement
- Update the problem statement to focus on transparency, visibility, and accountability.

#### 2.2 Professional Language
- Remove emotional language and use clear, simple, and professional wording.

#### 2.3 Responsibility Tracking
- Clearly mention who updates the request status at each stage and how responsibility is tracked.

#### 2.4 Simplified Workflow Explanation
- Simplify workflow descriptions so they are easy to understand for users and reviewers.

#### 2.5 Assumptions and Limitations
- Clearly state system assumptions, limitations, and boundaries.
